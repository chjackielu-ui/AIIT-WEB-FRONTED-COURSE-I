## 1. 项目整体架构

```
ms-main/
├── index.html              # 首页 — 项目入口
├── map.html                # 美食地图页 — SVG 交互地图
├── list.html               # 美食图鉴页 — 多维度筛选列表
```

**📍 功能 3：本地搜索模块**

> **知识点：** JavaScript `Array.filter()` + `Event: input`（实时搜索） + `Event: keypress`（回车搜索） + 模板字符串 + DOM动态渲染

**数据流：**
```
用户输入 → input/keypress/click事件 → performSearch()
→ foodBosses.filter(匹配算法) → 动态生成HTML → innerHTML渲染
```

| 按钮/元素 | 功能 | 背后实现 |
|-----------|------|----------|
| **搜索输入框** `#searchInput` | 实时搜索 | 监听 `input` 事件：长度≥1 → 自动执行搜索；长度=0 → 隐藏结果 |
| **搜索按钮** `#searchBtn` | 手动触发搜索 | 监听 `click` 事件 → `performSearch()` |
| **回车键** | 键盘触发搜索 | 监听 `keypress` 事件 → `e.key === 'Enter'` |
| **搜索结果项** | 点击跳转详情页 | `onclick="window.location.href='detail.html?id=${food.id}'"` — 通过URL传递ID参数 |

**匹配逻辑 (`performSearch` 函数)：**

```javascript
const results = foodBosses.filter(food => {
    return food.name.toLowerCase().includes(keyword) ||  // 美食名
           food.city.toLowerCase().includes(keyword) ||  // 城市
           food.region.toLowerCase().includes(keyword) || // 区域
           food.type.toLowerCase().includes(keyword) ||   // 口味类型
           food.taste.some(t => t.toLowerCase().includes(keyword)) || // 口味标签（数组）
           food.desc.toLowerCase().includes(keyword);     // 简介
});
```

> **涉及知识点：**
> - `Array.filter()` — 高阶函数过滤
> - `String.toLowerCase()` — 大小写不敏感匹配
> - `String.includes()` — ES6字符串包含判断
> - `Array.some()` — 数组中任一元素满足条件
> - 模板字符串 `` `...${}...` `` — 动态HTML生成

#### 3.2.2 SVG地图渲染核心流程 ⭐

> **核心知识点：** GeoJSON → 坐标投影 → SVG Path → DOM动态注入

**完整渲染管线：**

```
china_geo_data.js (GeoJSON)  →  initMap()  →  buildMap()
  ├─ 1. 定义投影参数 (地图边界 + SVG画布大小)
  ├─ 2. project(lng, lat)  经纬度 → SVG坐标 (线性映射)
  ├─ 3. geoToPath(geometry)  坐标点 → SVG d属性 (M/L/Z命令)
  ├─ 4. 为每个省份生成:
  │     ├─ <path>  (填充区域)
  │     └─ <text>  (省份名称标签)
  ├─ 5. innerHTML 注入容器
  └─ 6. bindEvents() 绑定交互事件
```

**① 数据源 (`china_geo_data.js`)：**

- 格式：`var CHINA_GEO = { "type": "FeatureCollection", "features": [...] }`
- 每个 feature 包含：
  - `properties.name`：省份名称
  - `properties.centroid`：视觉中心点坐标 `[lng, lat]`
  - `geometry`：边界坐标（类型为 `Polygon` 或 `MultiPolygon`）

**② 坐标投影算法 (`project` 函数)：**

```javascript
function project(lng, lat) {
    const x = ((lng - MAP_LNG_MIN) / (MAP_LNG_MAX - MAP_LNG_MIN)) * MAP_W;
    const y = ((MAP_LAT_MAX - lat) / (MAP_LAT_MAX - MAP_LAT_MIN)) * MAP_H;
    return [x.toFixed(2), y.toFixed(2)];
}
```

> **知识点：线性映射（Linear Mapping）**
> - 中国经度范围：73°E ~ 136°E → 映射到 0 ~ 1000 (SVG宽度)
> - 中国纬度范围：17°N ~ 54°N → 映射到 750 ~ 0 (SVG高度，Y轴反转)
> - 公式：`output = (input - input_min) / (input_max - input_min) * output_range`
> - 纬度需要反转（中国最北54°在SVG Y轴最小处）

**③ SVG 路径生成 (`geoToPath` 函数)：**

```javascript
function geoToPath(geometry) {
    // 处理 Polygon 和 MultiPolygon 两种类型
    // 使用 SVG 路径命令：
    //   M x,y  = MoveTo (移动到起点)
    //   L x,y  = LineTo (直线到点)
    //   Z      = ClosePath (闭合路径)
    // 生成格式: "M x1,y1 L x2,y2 L x3,y3 ... Z"
}
```

> **知识点：SVG Path 微语言**
> - `M` (MoveTo) — 不画线，只移动笔触
> - `L` (LineTo) — 从当前点画直线到目标点
> - `Z` (ClosePath) — 闭合当前路径
> - 省份形状的复杂度决定了路径中坐标点数量（约数百到数千个点）

**④ SVG 渐变定义（安徽省高亮）：**

```xml
<defs>
    <linearGradient id="anhuiGradient" x1="0%" y1="0%" x2="100%" y2="100%">
        <stop offset="0%" style="stop-color:#E63946;stop-opacity:1" />
        <stop offset="100%" style="stop-color:#9B2226;stop-opacity:1" />
    </linearGradient>
</defs>
```

> **知识点：SVG `<linearGradient>`** — 定义线性渐变，通过 `url(#anhuiGradient)` 引用

**⑤ 省份标签文字描边：**

```css
.map-label {
    paint-order: stroke fill;        /* 先描边后填充 */
    stroke: rgba(255, 255, 255, 0.85); /* 白色描边 */
    stroke-width: 2.5px;             /* 描边宽度 */
    stroke-linejoin: round;          /* 描边圆角连接 */
}
```

> **知识点：`paint-order`** — 控制SVG元素绘制顺序，`stroke fill` 表示先画描边再画填充，实现文字描边效果（文字在任意颜色背景上都清晰可读）

#### 3.2.3 地图交互系统 ⭐

> **核心知识点：** DOM事件（mouseenter/mousemove/mouseleave/click） + `getBoundingClientRect` + CSS `position: absolute`

| 交互 | 触发事件 | 实现逻辑 |
|------|---------|----------|
| **悬浮省份 → 高亮** | `mouseenter` | 省份 `<path>` 的CSS `:hover` 自动触发（`fill-opacity: 0.85` + 金色描边） |
| **悬浮省份 → Tooltip显示** | `mouseenter` | `tooltip.classList.add('show')`，更新省份名称和美食数量 |

| **移出省份 → Tooltip消失** | `mouseleave` | `tooltip.classList.remove('show')` |

**Tooltip 定位原理：**

```javascript
path.addEventListener('mousemove', function(e) {
    const rect = mapContainer.getBoundingClientRect();  // 获取地图容器相对视口位置
    const x = e.clientX - rect.left;                     // 鼠标相对容器X坐标
    const y = e.clientY - rect.top;                      // 鼠标相对容器Y坐标
    tooltip.style.left = x + 'px';
    tooltip.style.top = y + 'px';
});
```

> **知识点：`element.getBoundingClientRect()`** — 返回元素相对于视口的位置和尺寸，`e.clientX/clientY` 是鼠标相对于视口的坐标，两者相减得到相对于容器的坐标。

**Tooltip CSS 三角箭头（纯CSS）：**

```css

    border-color: rgba(255, 255, 255, 0.95) transparent transparent transparent;
}
```

> **知识点：CSS 三角形技巧** — 利用边框交汇处的斜角，设置一边有颜色其余透明，即形成三角形箭头。

```javascript
const processRing = ring => {

};

// MultiPolygon: 多个独立多边形（如浙江含岛屿）
```

### 4.4 今日BOSS确定性随机算法

> **核心知识点：** 伪随机种子 + 位运算 + 日期作为种子

```javascript
function getTodayBoss() {
    const today = new Date();
    const dateString = `${today.getFullYear()}-${today.getMonth()+1}-${today.getDate()}`;
    // 例如: "2026-6-14"

    // 基于日期字符串生成确定性种子
    let seed = 0;
    for (let i = 0; i < dateString.length; i++) {
        seed = ((seed << 5) - seed) + dateString.charCodeAt(i);
        seed = seed & seed;  // 转换为32位整数
    }

    const index = Math.abs(seed) % foodBosses.length;
    return foodBosses[index];
}
```

> **知识点详解：**
> - `new Date()` — 获取当前日期时间
> - `getFullYear() / getMonth() / getDate()` — 提取年月日
> - `dateString.charCodeAt(i)` — 获取字符ASCII码
> - `(seed << 5) - seed` — **左移5位**等价于 `seed * 31`，经典哈希乘法（如Java的 `hashCode`）
> - `seed & seed` — 位与运算，将数字转换为32位有符号整数（防止溢出）
> - `Math.abs(seed) % length` — 取绝对值后取模，映射到数组索引范围

**算法特性：**
- **确定性**：同一天 → 相同种子 → 相同索引 → 相同BOSS
- **均匀性**：日期字符串微小变化 → 种子大幅变化 → 均匀分布
- **无需存储**：纯计算，不需要后端、不需要LocalStorage

> **为什么叫"BOSS"？** 将美食比作游戏中的BOSS，每天需要挑战一个不同的美食，增加趣味性和游戏化体验。
