## 三、HTML 编写习惯

### 3.1 基本结构

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <!-- 内容 -->
</body>
</html>
```

**注意点：**
- `lang` 属性默认用 `"en"`，少数中文场景会用 `"zh-CN"`。
- `<title>` 经常是占位用的 `"Document"`，实际项目中请提醒改为有意义的标题。
- 缩进使用 **4 个空格**。
- 中文注释和内容混用很常见。

### 3.2 标签使用习惯

- 标题标签（h1-h6）喜欢配合 `align` 属性做居中。
- 段落文本喜欢用 `<br>` 换行，而非多个 `<p>`。
- 表格偏好设置 `border="1"` 并搭配内联样式做美化。
- 表单常用 `<fieldset>` + `<legend>` 包裹，结构清晰。
- 图片常用网络 URL（如学校官网图片），少用本地相对路径。

### 3.3 注释风格

```html
<!-- 双标签：h1-h6、p -->
<!-- 单标签：hr -->
<!-- 列表讲解 -->
<!-- 第一种清除浮动：overflow: hidden; -->
```

- 注释用中文，起到"笔记/讲义"的作用。
- 经常在注释中总结知识点，这既是给自己看的笔记，也是给同学看的参考。

## 四、CSS 编写习惯

### 4.1 使用方式偏好

**优先级：内嵌样式表 > 行内样式 > 外部样式表**

绝大多数文件使用 `<style>` 标签写在 `<head>` 中。少数演示"链入式"的文件才会使用外部 `.css` 文件。行内样式会在快速测试时使用，但注释会标注"不推荐此用法"。

### 4.2 选择器命名风格

**类名特点：短小、中文拼音缩写或英文缩写。**

| 实际命名 | 含义 | 风格类型 |
|----------|------|----------|
| `.mydiv` | 我的div | 英文简写 |
| `.tit2` | title 2 | 英文缩写+数字 |
| `.dllb` | 迪丽冷⑧ | 拼音首字母 |
| `.temp-clod` | temperature-cold | 英文连字符 |
| `.md`, `.md1`, `.md2` | margin div | 极短缩写+数字 |
| `.bor`, `.pad` | border, padding | 极短缩写 |
| `.f1`, `.f2`, `.f3`, `.f4` | font 1/2/3/4 | 字母+数字 |
| `.p1`, `.p2`, `.p3`, `.p4` | paragraph | 字母+数字 |
| `.box`, `.item`, `.son`, `.father` | 通用英文 | 常见命名 |
| `#dabox`, `#shang`, `#lg`, `#dh`, `#hz`, `#foot` | 大盒子/上/logo/导航/汇总/foot | 拼音缩写+英文 |

**命名规律总结：**
- 练习文件偏好 **极短缩写**（2-4 个字符）或 **字母+数字**。
- 实战作业会使用更有意义的拼音/英文命名。
- **不使用 BEM 命名法**，不使用 CSS 预处理器。

### 4.3 属性书写顺序

没有严格的书写顺序规范，但大致遵循：**定位 → 盒模型 → 字体 → 颜色 → 装饰**。

### 4.4 颜色值使用

- 偏好 `rgb()` 和 `rgba()` 函数形式。
- 也会使用十六进制（如 `#1547a0`）和具名颜色（如 `red`, `blue`）。
- 喜欢带透明度的颜色做半透明效果。

### 4.5 数值单位

- 绝大多数使用 `px`。
- 少数字体使用 `%` 或 `em`。
- 不会使用 `rem`、`vw`、`vh` 等现代单位。

### 4.6 CSS 注释

```css
/* 通配符选择器 *{}*/
/* 块级标签选择器 标签{}*/
/* 类选择器 .类名{}*/
/* id选择器 #id名{}*/
/* 第一种清除浮动：overflow: hidden; */
/* 垂直居中 */
```

- 注释用中文，标注"这是什么选择器/技巧"。
- 经常保留被注释掉的代码作为参考。

## 五、JavaScript 编写习惯

### 5.1 变量声明

当前阶段 **混用 `var` 和 `let`**，正在从 `var` 过渡到 `let`。尚未使用 `const`。

```javascript
var name = "西杰";
var age = 14;
let array1 = [2, 5, 2, "hello", false];
let date = new Date();
```

### 5.2 函数命名

```javascript
function fn() { ... }       // 简短通用名
function fn1() { ... }      // 数字编号
function test() { ... }     // 测试函数
```

- 练习文件的函数名非常简短（`fn`, `fn1`, `fn2`, `test`）。
- 实战作业中才会使用有意义的名字。

### 5.3 输出调试

```javascript
console.log("hello javascript");
console.log(typeof name);
console.log(array1);
```

- **大量使用 `console.log()`** 进行调试输出，几乎每个操作后都要打印确认。
- 早期文件也会使用 `document.write()`。

### 5.4 事件绑定

三种方式都会用，**学习进度从行内到监听器递增**：

```html
<!-- 方式1：行内式（最早学的） -->
<h1 onclick="fn()">今天周一</h1>

<!-- 方式2：动态绑定 -->
<script>
two.onclick = function() { ... };
</script>

<!-- 方式3：事件监听器（最晚学的，最新文件用的多） -->
<script>
three.addEventListener("click", function() { ... });
</script>
```

### 5.5 字符串处理

```javascript
var str = "helloJS";
var str3 = "哥，们，生，意，行，啊";
console.log(str.charAt(3));
console.log(str3.split(","));
console.log(year + "年" + month + "月" + day + "日");
```

- 使用 `+` 拼接字符串（尚未使用模板字符串）。
- 中文逗号分隔的字符串是常见测试数据。

### 5.6 注释风格

```javascript
// 控制台输出
// 页面输出
// var tag_sum_num = 100;
// for (var i = 1; i <= tag_sum_num; i++) { ... }
// 数组的常用方法
```

- 单行注释 `//`，中文说明。
- 经常保留被注释掉的旧代码作为参考。
